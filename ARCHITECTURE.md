# 🏗️ Архитектура базы данных Miss Kurochka

## 📊 Обзор

База данных спроектирована для PWA приложения сети ресторанов быстрого питания Miss Kurochka. Архитектура поддерживает:

- Управление несколькими филиалами
- Гибкую систему меню с модификаторами
- Обработку заказов (самовывоз и доставка)
- Интеграцию с платежной системой Finik
- Мультиязычность (русский, английский, кыргызский)
- Аналитику и статистику

## 🎯 Ключевые архитектурные решения

### 1. Мягкое удаление (Soft Delete)
Вместо физического удаления записей используется поле `deleted_at`:
- Сохраняется история операций
- Возможность восстановления данных
- Целостность связанных записей

**Примечание**: В текущей схеме Prisma мягкое удаление не реализовано на уровне схемы, но может быть добавлено через middleware.

### 2. Мультиязычность через JSONB
Поля с переводами хранятся в формате JSON:
```json
{
  "ru": "Куриные крылышки",
  "en": "Chicken wings",
  "ky": "Тоок канаттары"
}
```

**Преимущества**:
- Гибкость добавления новых языков
- Не требуется создание отдельных таблиц переводов
- Быстрый доступ к переводам

### 3. Гибкая система модификаторов
Модификаторы организованы в группы с типами выбора:
- **Single** - выбор одного варианта (например, соус)
- **Multiple** - выбор нескольких (например, дополнительные ингредиенты)

**Структура**:
```
ModifierGroup (Группа: "Выбор соуса")
  ├── ModifierOption (Кетчуп, +0 сом)
  ├── ModifierOption (Майонез, +0 сом)
  └── ModifierOption (Острый, +10 сом)
```

### 4. Иерархия категорий
Категории меню поддерживают вложенность через `parent_id`:
```
Курица
  ├── Крылышки
  ├── Наггетсы
  └── Стрипсы
```

### 5. Геолокация
Координаты хранятся в формате `DECIMAL`:
- `latitude` - DECIMAL(10,8) - широта
- `longitude` - DECIMAL(11,8) - долгота

Зоны доставки используют GeoJSON полигоны в JSONB.

## 📦 Блоки системы

### 👤 Пользователи и авторизация

#### users
Основная таблица пользователей с тремя ролями:
- **admin** - администратор системы
- **branch** - сотрудник филиала
- **customer** - клиент

**Особенности**:
- Email и username опциональны (для гостевых заказов)
- Хранится хэш пароля (bcrypt/argon2)
- Поддержка аватаров

### 🏢 Филиалы

#### branches
Информация о филиалах сети:
- Контактные данные
- Геолокация
- Статус работы
- Временное закрытие

#### branch_schedules
График работы по дням недели (0=Понедельник, 6=Воскресенье)

#### branch_users
Связь сотрудников с филиалами (many-to-many)

### 🍔 Меню

#### menu_categories
Категории меню с поддержкой:
- Иерархии (parent_id)
- Привязки к филиалу (для уникального меню)
- Мультиязычности

#### menu_items
Позиции меню с полной информацией:
- Цена и себестоимость
- Вес/объем
- Время приготовления
- Пищевая ценность (калории, БЖУ)
- Аллергены
- Уровень остроты
- Флаги (вегетарианское, веганское, новинка, хит)

#### menu_item_images
Фотографии блюд с флагом основного изображения

#### stop_list
Временно недоступные блюда:
- Причина недоступности
- Ожидаемое время возврата
- Кто остановил/восстановил

### ⚙️ Модификаторы

#### modifier_groups
Группы модификаторов:
- Тип выбора (single/multiple)
- Обязательность
- Мин/макс количество выборов

#### modifier_options
Варианты модификаторов:
- Изменение цены (delta)
- Флаг по умолчанию

#### menu_item_modifiers
Связь блюд с группами модификаторов

### 🧾 Заказы

#### orders
Основная информация о заказе:
- Уникальный номер
- Тип (самовывоз/доставка)
- Статус (pending → confirmed → preparing → ready → completed)
- Способ оплаты
- Временные метки (готов, доставлен, отменен)

#### order_items
Позиции в заказе:
- Количество
- Цена на момент заказа (фиксируется)
- Комментарий к позиции

#### order_item_modifiers
Выбранные модификаторы для каждой позиции

### 💳 Платежи

#### payments
Платежи по заказам:
- Способ оплаты
- Сумма и валюта
- Статус
- ID транзакции Finik

**Особенности**:
- Один заказ может иметь несколько платежей (частичная оплата)
- Поддержка возвратов

### 🚚 Доставка

#### delivery_zones
Зоны доставки:
- Полигон в формате GeoJSON
- Стоимость доставки
- Минимальная сумма заказа
- Время доставки

#### delivery_addresses
Адреса клиентов:
- Полный адрес
- Детали (квартира, подъезд, этаж, домофон)
- Координаты
- Флаг адреса по умолчанию

### 🎯 Маркетинг

#### banners
Баннеры и акции:
- Заголовок и подзаголовок
- Изображение
- Ссылка
- Период действия
- Порядок отображения

### 📊 Аналитика

#### popular_items_stats
Статистика популярных блюд:
- Количество заказов
- Проданное количество
- Выручка
- Агрегация по дням

**Использование**:
- Топ блюд за период
- Анализ продаж
- Прогнозирование спроса

### ⚙️ Настройки

#### system_settings
Системные настройки в формате key-value:
```json
{
  "key": "app.currency",
  "value": {
    "code": "KGS",
    "symbol": "сом"
  }
}
```

## 🔗 Связи между таблицами

### One-to-Many (1:N)
- Branch → MenuCategory
- Branch → Order
- MenuCategory → MenuItem
- MenuItem → MenuItemImage
- Order → OrderItem
- OrderItem → OrderItemModifier

### Many-to-Many (N:M)
- Branch ↔ User (через branch_users)
- MenuItem ↔ ModifierGroup (через menu_item_modifiers)

### Self-referencing
- MenuCategory → MenuCategory (parent_id)

## 🔐 Безопасность

### Индексы
Prisma автоматически создает индексы для:
- Primary keys (id)
- Foreign keys
- Unique constraints

### Каскадное удаление
Настроено для:
- Удаление филиала → удаление графика работы
- Удаление заказа → удаление позиций заказа
- Удаление позиции → удаление модификаторов

### Валидация
На уровне приложения:
- Проверка email
- Валидация телефона
- Проверка цен (> 0)
- Валидация количества

## 📈 Масштабирование

### Оптимизация запросов
```typescript
// Плохо - N+1 запросов
const items = await prisma.menuItem.findMany()
for (const item of items) {
  const images = await prisma.menuItemImage.findMany({
    where: { menuItemId: item.id }
  })
}

// Хорошо - 1 запрос с include
const items = await prisma.menuItem.findMany({
  include: { images: true }
})
```

### Пагинация
```typescript
const items = await prisma.menuItem.findMany({
  skip: (page - 1) * pageSize,
  take: pageSize,
})
```

### Кэширование
Рекомендуется кэшировать:
- Меню (обновляется редко)
- Список филиалов
- Системные настройки

## 📝 Best Practices

### 1. Используйте транзакции для связанных операций
```typescript
await prisma.$transaction(async (tx) => {
  const order = await tx.order.create({ data: orderData })
  await tx.payment.create({ data: paymentData })
})
```

### 2. Фиксируйте цены в заказах
```typescript
// Сохраняем цену на момент заказа
const order = await prisma.order.create({
  data: {
    items: {
      create: {
        menuItemId: item.id,
        unitPrice: item.price, // Фиксируем текущую цену
        quantity: 2,
      }
    }
  }
})
```

### 3. Используйте select для оптимизации
```typescript
// Получаем только нужные поля
const users = await prisma.user.findMany({
  select: {
    id: true,
    fullName: true,
    email: true,
  }
})
```

### 4. Обрабатывайте ошибки
```typescript
try {
  await prisma.order.create({ data })
} catch (error) {
  if (error.code === 'P2002') {
    // Unique constraint violation
  }
}
```

## 🔄 Жизненный цикл заказа

```
pending (создан)
    ↓
confirmed (подтвержден филиалом)
    ↓
preparing (готовится)
    ↓
ready (готов к выдаче/доставке)
    ↓
delivering (доставляется) - только для delivery
    ↓
completed (завершен)

cancelled (отменен) - возможен на любом этапе
```
